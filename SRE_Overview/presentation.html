What is Site Reliability Engineering (SRE)?

When I first entered IT there were 2 groups

Create Software

Features and bug fixes

Throw the code over the wall

Limited or no access in prod-like environments

Provision nmew systems

Deploy & update applications

Monitoring, incident managment etc

The Blame Game

This lead to a culture of name and shame

Google created the concept of Site Reliability Engineering in 2004 because traditional Sysadmins don't scale

The idea? Treat Ops like a software engineering problem

50/50 split between operations and engineering tasks

At the end of the day, no one actually cares about the platform

The focus on SRE is all about the end user experience

Traditional Operations KPIs (mostly) no longer apply

reorients the focal point away from individual systems and performance towards the goal of making end user happy

A Scaling Problem

TOIL

Error budgets are an important concept in SRE.

In essence, once you figure out what constitutes an error for an application you can "spend" errors to make improvements

Availability is expensive; Errors are ok

SRE can indeed be "magical" if done properly

It can drastically increase technological readiness and "time to market"

It might even enable you to set binding SLAs in place

"Just Enough" SRE: Requirements
  • Focus on delivering application stability
  • Error budget/observability
  • Incident management/learning from incidents
  • Blameless Post Mortems
  • Ability to push back/deny deployment
  • Communication / Collaboration between teams

Context Switching Is Bad...

In a multitasking context, it refers to the process of storing the system state for one task, so that task can be paused and another task resumed.

A context switch can also occur as the result of an interrupt, such as when a task needs to access disk storage, freeing up CPU time for other tasks

In human terms, someone walking up to your desk, stopping to look at a notification or other distractions are forms of context switching

Context Switching Is Super Bad...

Our brains aren't made to multi-task
